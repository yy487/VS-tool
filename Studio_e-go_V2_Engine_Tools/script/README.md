# script

这里放 `Studio_e-go_V2` 的正式脚本实现。

当前按两层语义组织：

- 外层容器层
  - `BtText.dat`
  - `Script.dat`
  - `*.scr`
  - 以及它们内部的 `TSCR` / `TUTA` / `TCRP` / `TXT0` / `M3H0`
- 内层指令层
  - 真正的 `op + arg` 脚本主体

当前已经进入正式实现的是外层容器层：

- `BtText.dat`
  - 原始文件头是 `TSCR`
  - 文件头后是可逆的 mode-5 字交换异或载荷
  - 载荷解码后是 `TUTA`
  - 其中已确认存在 `TXT0` 索引字符串池
- `*.scr`
  - 原始文件头是 `SCR `
  - 文件头后是可逆的 mode-2 载荷
  - 载荷解码后已能稳定看到字符串字面量和后续内层脚本载荷

当前阶段方向不是继续优先追完整内层指令层，而是：

- 以文本主载体为主
- 以可变长文本处理为主
- 以自洽回写为主

内层指令层尚未建立正式反编译 / 回编链，当前也不作为这一阶段默认阻塞项。
